# SEO Chile — Plan de implementación

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Posicionar simcii.ai en Google Chile para búsquedas de categoría mediante 7 páginas estáticas de solución/industria + contenido rastreable en la landing.

**Architecture:** Páginas HTML estáticas puras (sin React/Three.js/Mapbox) que comparten el CSS de la landing (extraído a `assets/simcii.css`). URLs limpias vía rewrites de Vercel. La landing gana contenido estático dentro de `#root` que React reemplaza al hidratar.

**Tech Stack:** HTML/CSS estático, React 18 UMD (solo landing), Babel (`npm run build`), Vercel.

**Spec:** `docs/superpowers/specs/2026-07-21-seo-chile-design.md`

## Global Constraints

- Directorio de trabajo: raíz del repo (`SIMCII - WEB PAGE/`).
- Tono de copy (DESIGN.md): "autoridad tranquila" — evidencia, sin hype; verbos medir/leer/anticipar. PROHIBIDO: "revolucionario/disruptivo/mágico", jerga de IA (cerebros, redes neuronales), promesas sin respaldo.
- Tipografía/paleta: solo tokens CSS existentes (`--paper`, `--bg`, `--surface`, `--azul-claro`, `--terracota`, etc.). Sin itálica. Énfasis por color azul cívico + peso.
- Páginas nuevas: CERO JavaScript (ni React ni analytics nuevos). Solo HTML + CSS.
- Todo texto en español de Chile. Dominio canónico: `https://simcii.ai`.
- Titles ≤ 60 caracteres, meta descriptions ≤ 155 caracteres, únicos por página.
- No clonar párrafos entre páginas (contenido duplicado penaliza).
- Commits frecuentes, uno por tarea como mínimo.
- Verificación: no hay framework de tests; cada tarea define comandos `grep`/`node`/`curl` concretos con salida esperada.
